Medallion Architecture: The Lakehouse Foundation
The Databricks-pioneered data organization pattern that transformed enterprise data infrastructure — structuring data into three progressive quality layers (Bronze, Silver, Gold) on open Delta Lake storage, unifying analytics, ML, and agentic AI on a single governed foundation that replaces the legacy two-platform (warehouse + lake) architecture.

What is the Medallion Architecture?

The Medallion Architecture is Databricks' recommended pattern for organizing data within a Lakehouse — structuring it into three progressive quality layers, each built on Delta Lake's ACID transaction foundation. Unlike traditional data architecture patterns that required separate physical systems for raw storage and analytics, the Medallion Architecture runs all three layers on the same cloud object storage (S3, ADLS, GCS) — governed by Unity Catalog, processed by Spark and Photon, and accessible to analytics, ML, and AI workloads simultaneously.

Each Medallion layer transition is an incremental, idempotent pipeline step — enabling efficient reprocessing when upstream data changes, full audit trails (Delta time travel), and regulatory compliance (GDPR deletion via Deletion Vectors propagates through all layers). The architecture has become the dominant enterprise data organization standard, adopted by 15,000+ Databricks customers and referenced by Snowflake, Microsoft, and Google as the benchmark data layer pattern.

The Three Layers
Layer 1
Bronze — Raw Ingestion
Exact copy of source data — preserving source fidelity for reprocessing and audits. Lakeflow Connect manages ingestion from 50+ connectors with automatic CDC and schema drift handling. No transformation applied: if source data is wrong, Bronze captures the error so upstream systems can be diagnosed. Delta Lake transaction log provides complete ingestion history. Retention policy governed by Unity Catalog.
Layer 2
Silver — Validated & Integrated
Cleaned, deduplicated, joined, and conformed data — the enterprise-wide single source of truth. Lakeflow Pipelines enforce data quality via EXPECT clauses (reject or quarantine records failing business rules). Cross-domain joins happen here: customer records merged with transaction data, product catalog aligned with inventory. The Silver layer is the foundation for all ML feature engineering and the input for downstream Gold aggregations.
Layer 3
Gold — Business-Ready
Aggregated, domain-specific tables consumed directly by BI tools, ML Feature Store, and AI agents. Gold tables are optimized for query performance via Photon's vectorized engine and Liquid Clustering. Genie enables natural language querying of Gold tables by business users. Agent Bricks agents read Gold layer data for decision-making and write results back to Lakebase for operational consumption. Unity Catalog lineage traces every Gold metric to its Bronze source automatically.
The Lakehouse vs. Legacy Two-Platform Architecture

The traditional enterprise data architecture required two separate systems: a data lake (cheap, flexible, ungoverned — Hadoop or S3) for raw storage and a data warehouse (structured, reliable, expensive — Teradata, Redshift, Snowflake) for analytics. This created three chronic problems: (1) data duplication — the same data lived in both systems with ETL pipelines synchronizing them; (2) governance fragmentation — different access controls, lineage tools, and compliance processes for each system; (3) ML exclusion — data scientists needed to extract data from the warehouse into a separate ML environment, creating a third copy with its own staleness and governance risks.

The Databricks Lakehouse collapses all three systems into one: Delta Lake provides warehouse-grade ACID reliability directly on cloud object storage (eliminating the need for a separate data warehouse), and Unity Catalog governs the entire stack — tables, ML models, AI agents, and files — with one access control plane. The result: no data duplication, no ETL pipelines between analytics and ML, and a single governance layer satisfying HIPAA, GDPR, SOX, and EU CSRD simultaneously.

2026 Extension: Beyond Tables
ML Models as Medallion Objects
Unity Catalog Model Registry extends the Medallion governance pattern to ML models: Bronze (experimental runs in MLflow), Silver (validated challenger models), Gold (production champion models serving inference). Every model version is a Unity Catalog object with lineage tracing back to the training dataset's Silver table — enabling model audits required for regulated industries.
Agent Bricks State in Lakebase
Agent Bricks agents persist operational state, logs, and decisions to Lakebase (PostgreSQL-compatible OLTP) — which bidirectionally syncs to Delta Lake Gold tables. This means every agent action is captured as a structured, queryable Gold-layer record: full operational AI auditability without a separate logging infrastructure.
AI-First Silver Transformations
Lakeflow's AI-first ETL functions (ai_classify(), ai_extract(), ai_summarize()) run directly in Silver-layer DLT pipelines — enabling document parsing, entity extraction, and sentiment tagging as native pipeline steps. Unstructured data (PDFs, emails, support tickets) becomes structured Silver-layer records without separate ML microservices.
Unity Catalog Lineage Across All Layers
Unity Catalog automatically tracks column-level lineage across all three Medallion layers — zero manual annotation required. A compliance audit requiring evidence of where a patient record flowed from Bronze ingestion to Gold dashboard can be answered in a Unity Catalog UI query, not a multi-week manual investigation.
