What is Photon?

Photon is Databricks' proprietary native query engine, built entirely in C++ and deeply integrated with Delta Lake. It replaces the JVM-based Apache Spark execution engine for SQL and DataFrame workloads, using modern CPU hardware features — SIMD vectorization, advanced branch prediction, and cache-friendly memory layouts — to dramatically accelerate query performance without any changes to user code or SQL queries.

Photon operates transparently: the same SQL or PySpark code runs on Photon automatically when it is enabled on a cluster, with Spark serving as the fallback for operations Photon does not yet support. Because Photon is deeply co-designed with Delta Lake's columnar Parquet storage, it achieves optimal performance specifically on Delta tables — the format used throughout the Databricks Medallion Architecture. Photon is included in Databricks Premium and above at no additional charge, making it the default performance layer for all SQL and ETL workloads.

Key Technical Capabilities
SIMD Vectorized Execution
Photon processes columnar data in batches using CPU SIMD (Single Instruction, Multiple Data) instructions — operating on 512 bits of data per clock cycle vs. one row at a time in the JVM model. For aggregation-heavy analytical queries, this is the primary source of 3–4× speedup.
Delta Lake Native I/O
Photon reads Delta Lake Parquet files using its own optimized I/O path — bypassing the JVM deserialization pipeline and working directly with the columnar byte layout on disk. Combined with Predictive Optimization and Liquid Clustering, this eliminates nearly all unnecessary I/O.
Adaptive Query Execution Integration
Photon integrates with Spark's Adaptive Query Execution (AQE) to dynamically reoptimize query plans at runtime based on actual data statistics. This means Photon benefits from both static compile-time optimization and dynamic runtime adaptation — a combination unavailable in competing engines.
Transparent Compatibility
Zero code changes required. Photon accelerates SQL, DataFrame API (PySpark, Scala, Java), and Delta Live Tables automatically. Unsupported operations fall back to JVM Spark silently. New Photon capabilities are added with every release, expanding coverage of the Spark operation surface.
Serverless Compute Powered by Photon
Databricks Serverless compute (notebooks, workflows, DLT) runs on Photon by default. When customers adopt serverless — as Databricks strongly encourages for cost efficiency — they also automatically get Photon's performance benefits without managing instance types or cluster sizes.
Supported Operation Coverage
Photon accelerates: SELECT with filters and projections, GROUP BY aggregations, JOIN operations (hash, sort-merge, broadcast), ORDER BY with top-k optimization, window functions, and Delta Lake MERGE/UPDATE/DELETE. Coverage expands with each platform release.

Photon's impact increases at higher Medallion layers where transformations become more aggregation-heavy and JOIN-intensive. In the Bronze layer (raw ingestion), I/O and network are typically the bottleneck; Photon provides modest gains. In the Silver and Gold layers — where complex JOINs, window functions, and GROUP BY aggregations dominate — Photon's vectorized engine delivers its maximum 3–4× speedup. For Gold-layer pipelines that feed BI dashboards or ML feature stores, Photon's acceleration directly translates to fresher data and faster time-to-insight.

Photon is Databricks' answer to the "but isn't Snowflake faster for SQL?" objection. For SQL-centric BI workloads on well-structured data, Snowflake's engine is highly optimized.
