What is Lakeflow?

Lakeflow is Databricks' answer to the fragmented, multi-tool data engineering stack that enterprises have accumulated over the past decade — separate ingestion tools, transformation frameworks, orchestration schedulers, and monitoring dashboards that don't share governance, lineage, or operational context. Lakeflow unifies these into a single platform: from raw source connectivity through pipeline orchestration to AI-enriched data delivery, all governed by Unity Catalog and observable through a single operational pane.

Lakeflow is built on three pillars: Lakeflow Connect (managed ingestion connectors that pull data from SaaS, databases, and event streams into Delta Lake), Lakeflow Pipelines (the declarative Delta Live Tables engine that automatically manages data quality, incremental processing, and lineage), and Lakeflow Designer (GA April 2026 — a drag-and-drop visual pipeline builder that makes data engineering accessible to a broader audience). Together, they cover the full data engineering lifecycle on a single unified stack.

Three Pillars of Lakeflow
🔌
Lakeflow Connect
Managed ingestion connectors for 50+ SaaS sources (Salesforce, SAP, ServiceNow), databases (SQL Server, Oracle, MySQL), and event streams (Kafka, Kinesis). Automatically handles schema drift, CDC (Change Data Capture), and initial full loads — eliminating the operational burden of custom connector maintenance.
⚙️
Lakeflow Pipelines
Built on Delta Live Tables (DLT) — Databricks' declarative pipeline framework. Data engineers define what data should look like (expectations, transformations) rather than how to compute it. DLT handles incremental processing, dependency ordering, data quality enforcement (EXPECT clauses), and Unity Catalog lineage automatically.
🎨
Lakeflow Designer
GA April 2026 — a drag-and-drop visual interface for building Lakeflow pipelines without writing code. Analysts and data engineers can visually connect sources, define transformations, and schedule workflows. Generates production-quality DLT Python/SQL code that can be promoted to source control. Designed to accelerate pipeline development for non-expert practitioners.
AI-First Data Engineering

Lakeflow's most distinctive 2026 capability is its native AI integration: data engineers can embed AI directly into ETL pipelines without building separate AI microservices. Lakeflow AI functions allow pipelines to call Foundation Model APIs (GPT-5.5, Claude, Llama, Gemini) directly within SQL transformations — enabling document parsing, entity extraction, sentiment classification, and multimodal analysis as first-class pipeline steps.

AI Functions in Pipelines
Call Foundation Model APIs directly within DLT SQL transformations: ai_classify(), ai_extract(), ai_summarize(). Pipelines can parse invoices, classify support tickets, extract entities from unstructured text — all within the same Medallion Architecture flow, governed by Unity Catalog.
Genie Code for Pipelines
Genie Code assists data engineers in authoring, explaining, and troubleshooting Lakeflow pipelines in natural language. Engineers can ask "Why is this pipeline failing?" or "Generate a Silver layer join between these two tables" — reducing time spent debugging and accelerating development cycles.
Agent Bricks Integration
Lakeflow pipelines can be triggered and orchestrated by Agent Bricks autonomous agents. An agentic replenishment system, for example, can trigger a Lakeflow pipeline to enrich demand data with external signals — then pass the enriched dataset to the next agent step without manual handoffs.
Unity Catalog Lineage
Every Lakeflow pipeline step is automatically tracked in Unity Catalog's column-level lineage graph. From source ingestion through Bronze → Silver → Gold, data engineers and compliance teams can trace exactly which source data produced which analytical output — zero manual annotation required.
